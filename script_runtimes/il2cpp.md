# IL2CPP

https://docs.unity3d.com/2022.3/Documentation/Manual/IL2CPP.html

## 编译步骤
1. Roslyn 编译 C# 到 DLL
2. 字节码裁剪，Unity linker 分析无法到达的类和函数
3. IL2CPP 转换 IL 到 C++
4. 编译生成的 C++ 代码、IL2CPP 的运行时部分（libil2cpp）
5. 根据目标平台，生成可执行文件或 DLL

## 运行时

https://github.com/4ch12dy/il2cpp/blob/master/unity_2020_x/libil2cpp/il2cpp-api-functions.h

IL2CPP 运行时（即 libil2cpp）是一个为支撑转换后的 C++ 代码而存在的“服务大礼包”，为游戏提供完整的虚拟机环境。其核心功能主要由以下几个部分组成：

- 🧠 类型与反射：维护着所有类型的完整元数据，这是反射（Reflection）功能的基础，并提供了在非托管环境中访问和操作这些元数据的内部 API。
- 📦 内存管理：包含整套 GC 机制（目前主要集成 Boehm GC），并负责统一管理内存的分配与释放。开发者依然可以通过 new 或 GC.Collect() 来影响 GC 行为。
- 🔧 代码校验：在生成的 C++ 代码中插入可选的运行时安全校验。开发者可通过 Il2CppSetOption 属性，为程序集、类或方法，精细化控制“数组越界”、“空引用”和“除零”校验的开启或关闭，以在安全性与性能间权衡。
- ⚙️ 平台抽象：提供与操作系统交互的统一抽象层，让 Unity 脚本能透明运行在各大平台，极大提升了跨平台能力。具体包括：
- 线程管理：封装了平台底层的线程 API，并为每个托管线程维护必要的运行时状态。IL2CPP 要求，任何需要与托管代码交互的原生线程都必须先“附加（Attach）”到运行时。
- 文件 I/O 与系统调用：对文件读写、网络请求等底层操作进行了统一封装。
- 🛠️ 其他关键功能：还包括异常处理（支持 C# 的 try-catch-finally 语义）、P/Invoke 互操作、内部调用（在 C++ 端直接高效操作托管对象）、调试支持（配合 Unity 编辑器进行断点、变量查看等）以及多种性能优化（如：方法内联、死代码消除、代码剥离和泛型代码共享等）。
